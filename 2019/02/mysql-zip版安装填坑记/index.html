Mysql zip版安装填坑记

二月 16, 2019 发布在 mysql

一、Mysql下载

进入官网，这里选择的版本是5.6.44，版本可自行选择。下载完成之后，解压到某个文件夹，我的是E:\software\mysql-5.6.44-winx64。

二、配置环境变量

这一步其实可有可无，配完环境变量的好处是我们可以在任意位置执行mysql的命令，而不配置环境变量则只能在mysql安装目录下的bin目录下执行mysql的命令。

配置环境变量的方法：

将mysql安装目录下的bin目录的全路径放到用户变量的path变量中去。我这里是E:\software\mysql-5.6.44-winx64\bin，在用户变量中的path变量追加;E:\software\mysql-5.6.44-winx64\bin即可。

三、遇到的问题

由于是解压安装，所以在整个安装过程中，我并没有像二进制安装那样输入任何的用户名和密码，于是我猜想mysql解压安装默认是没有密码的。
然而事实是： 所以mysql肯定是有一个初始密码的。

四、修改配置文件

在经过搜索引擎的帮助之后我发现，解压安装mysql是需要修改配置文件的。
进入mysql安装目录：E:\software\mysql-5.6.44-winx64，新建一个my.ini文件，内容如下：(其中basedir和datadir需要替换成自己的)

[mysql]
default-character-set=utf8

[mysqld]
basedir = E:/software/mysql-5.6.44-winx64
datadir = E:/software/mysql-5.6.44-winx64/data
port = 3306
character-set-server = utf8
default-storage-engine = INNODB

五、安装mysql

以管理员身份运行cmd，进入mysql的bin目录下，执行以下命令：

mysqld install

安装成功之后启动：

net mysql start

然后执行

mysqld --initialize-insecure

生成一个root用户名，空密码的用户数据

验证是否安装成功(无需密码)：

mysql -u root -p

修改密码：

mysql> set password for root@localhost = password('新密码');

六、mysql基本命令

对数据库的操作：

show databases;  //查看所有数据库
create database <数据库名>;
drop database <数据库名>;
use database;

对表的操作：

create table user(
    id int(4) not null primary key auto_increment,
    name char(20) not null,
    sex int(4) not null default '0',
    degree double(16,2)
);

//主键和外键
CREATE TABLE Skills (
   Id INTEGER PRIMARY KEY,
   SkillName CHAR(40),
   Description CHAR(100) 
);
CREATE TABLE EmployeeSkills (
   EmployeeID INTEGER NOT NULL,
   SkillID INTEGER NOT NULL,
   SkillLevel INTEGER NOT NULL,
   PRIMARY KEY( EmployeeID ),
   FOREIGN KEY (SkillID) REFERENCES Skills ( Id )
)

drop table <表名>;
desc <表名>; //查看表结构

//增删改查
INSERT INTO table_name(field1, field2,...fieldN)
VALUES(value1, value2,...valueN)

DELETE FROM table_name [WHERE ...]

UPDATE table_name SET field1=new-value1, field2=new-value2 [WHERE...]

SELECT column_name,column_name
FROM table_name
[WHERE Clause]
[LIMIT N][ OFFSET M]
